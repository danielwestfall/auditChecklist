  • Language attribute: Is the `lang` attribute used in the HTML code to specify the language of the page (e.g., ``)?

This means that if you use different languages on the same page, you should label each part with its language.
Check for:
  • Multiple languages: Are there any sections of text in different languages?
  • Language tags: Are language tags used to identify the language of each section (e.g., `Hola`)?

This means that if you use any uncommon words, jargon, or idioms, you should provide a way for people to understand what they mean.
Check for:
  • Unusual words: Are there any words or phrases that might be unfamiliar to some users?
  • Definitions: Are definitions provided (e.g., through a glossary, tooltips, or links to explanations)?

This means that if you use abbreviations, you should explain what they stand for.
Check for:
  • Abbreviations: Are any abbreviations used?
  • Expansions: Are the abbreviations expanded or explained (e.g., on first use, through a glossary, or using the `abbr` tag in HTML)?

This means that if the text is very complex, you should provide a simpler version or additional support for people who might find it difficult to read.
Check for:
  • Complex text: Is the text written in a way that's easy to understand?
  • Simpler version: Is there a simpler version available?
  • Support: Are there tools or resources to help people understand the text?

This means that if there are words that could be pronounced in different ways and have different meanings, you should provide a way for people to know the correct pronunciation.
Check for:
  • Ambiguous words: Are there any words that could be pronounced in multiple ways with different meanings?
  • Pronunciation guide: Is there a way to indicate the correct pronunciation (e.g., using phonetic transcription or audio clips)?
