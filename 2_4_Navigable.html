Navigable


This means that if the same content appears on lots of pages (like a header or menu), there should be a way to skip over it. This is helpful for people who use screen readers, as it saves them time.
Check for:
  • Repeated content: Is there any content that's repeated on many pages (e.g., header, navigation menu)?
  • Skip links: Are there "skip links" or other ways to bypass the repeated content?
  • Screen reader test: Does a screen reader announce a way to bypass the repeated content?

This means that every page should have a title that tells you what the page is about. This helps people understand where they are on the website.
Check for:
  • Page titles: Does each page have a title?
  • Descriptive titles: Do the titles accurately describe the page's content or purpose?
  • Unique titles: Are the titles different for different pages?

This means that when you tab through the page with your keyboard, the order of the elements should make sense.
Check for:
  • Tab order: Use the Tab key to move through the page. Does the focus move in a logical order?
  • Meaning and operation: Does the tab order affect how you understand or use the page?

This means you should be able to understand what a link does just by reading the link text or the text around it.
Check for:
  • Link text: Does the link text describe where the link goes or what it does?
  • Context: If the link text alone isn't clear, does the surrounding text provide enough information?
  • Avoid vague links: Avoid links like "click here" or "read more."

This means there should be more than one way to find a page on the website, like a search bar, site map, or breadcrumb trail.
Check for:
  • Navigation options: Are there multiple ways to navigate the website (e.g., menus, search, site map)?

This means that headings and labels should clearly describe the content they relate to.
Check for:
  • Headings: Do headings accurately describe the sections they introduce?
  • Labels: Do labels clearly identify the purpose of form fields or other elements?

This means that when you tab through the page with your keyboard, you should be able to see where the focus is.
Check for:
  • Keyboard navigation: Use the Tab key to move through the page. Is there a clear visual indicator showing which element has focus?

This means that the website should provide a way for you to know where you are within the site's structure.
Check for:
  • Breadcrumbs: Are there breadcrumbs showing the page hierarchy?
  • Navigation menus: Does the current page have a visual indicator in the navigation menu?
  • Headings: Are headings used to provide context about the page's location within the content?

This means using different levels of headings (like H1, H2, H3) to structure the content and make it easier to understand.
Check for:
  • Heading hierarchy: Are headings used in a logical order (H1, then H2, then H3, etc.)?
  • Screen reader test: Does a screen reader correctly identify the heading levels?
