WCAG 2.2 Guideline 1.3 Checklist (Beginner)


This means using headings, lists, and other formatting to organize content so people can understand how it's connected.
Check for:
  • Headings: Are headings used to organize sections of content?
  • Lists: Are lists used for items that belong together?
  • Layout: Is the content organized in a way that makes sense?
  • Screen readers: Does a screen reader present the information in a logical order?

This means putting content in an order that makes sense, so people aren't confused.
Check for:
  • Reading order: Is the content presented in the order it should be read?
  • Screen readers: Does a screen reader read the content in the correct order?
  • Keyboard navigation: Can you tab through the content in a logical order?

This means using more than just color to convey information, so people who can't see color can still understand it.
Check for:
  • Color: Is color the only way to tell things apart?
  • Alternatives: Are there other ways to understand the information, like text labels, patterns, or shapes?

This means the website or app should work whether you're holding your phone sideways or upright.
Check for:
  • Rotate device: Rotate your phone or tablet. Does the content still display correctly?
  • Content: Is any content cut off or overlapping?
  • Functionality: Do all the buttons and links still work?

This means using labels to tell people what information to put in each form field.
Check for:
  • Labels: Does each form field have a clear label?
  • Instructions: Are there instructions if needed?
  • Screen readers: Does a screen reader correctly identify the purpose of each field?

This means telling people if they make a mistake in a form and helping them fix it.
Check for:
  • Error messages: Are there clear error messages?
  • Error location: Is the error highlighted or easy to find?
  • Instructions: Are there instructions on how to fix the error?
