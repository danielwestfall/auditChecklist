WCAG 2.2 Guideline 4.1 Checklist (Beginner)


This means that the website's code should be written correctly, following the rules of the language it's written in (like HTML). This helps assistive technologies understand the code and present it correctly to users.
Check for:
  • Validator: Use an HTML validator to check the code for errors. (Search for "HTML validator" online.)
  • Error messages: Does the validator report any errors related to parsing or code structure?

This means that the website's code should clearly define the purpose and function of interactive elements (like buttons, links, and form fields), so assistive technologies can understand them and interact with them correctly.
Check for:
  • Inspect code: Use your browser's developer tools to inspect the code of interactive elements. Are the name, role, and value attributes used correctly?
  • Screen reader test: Does the screen reader correctly identify and interact with the elements?
