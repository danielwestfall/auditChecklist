WCAG 2.2 Guideline 1.2 Checklist (Beginner)


For live audio or video, provide a text alternative explaining what's happening.
Check for:
  • Live audio: Provide a text transcript or description of what's being said.
  • Live video: Provide a text description of what's being shown.

Captions are text versions of the spoken words in a video, displayed on the screen.
Check for:
  • Captions: Are there accurate captions synchronized with the audio?

Audio descriptions explain important visual details that aren't in the audio.
Check for:
  • Audio descriptions: Are there audio descriptions for important visual information?
  • Media alternative: If audio descriptions aren't possible, is there a text or audio version that provides the same information?

Provide real-time captions for live video content.
Check for:
  • Live captions: Are there accurate, real-time captions synchronized with the live audio?

Use extended audio descriptions if there's not enough time in the regular audio to describe all the important visuals.
Check for:
  • Extended audio descriptions: If there are long pauses in the audio, are extended audio descriptions used to explain visuals?

Provide sign language interpretation for videos where spoken audio is important.
Check for:
  • Sign language: Is there a sign language version of the video?

If pauses in the audio are too short, provide a separate video with extended audio descriptions, or a text alternative.
Check for:
  • Extended audio description or media alternative: If regular audio descriptions don't work, is there a separate version or a text alternative?

Provide real-time sign language interpretation for live video content.
Check for:
  • Live sign language: Is there real-time sign language interpretation available?

Provide a media alternative for audio only content.
Check for:
  • Transcript: Is there a transcript for all audio content?
