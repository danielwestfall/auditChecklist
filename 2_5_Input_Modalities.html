Input Modalities


This means that if you can do something on the website using complex gestures (like pinching or swiping), you should also be able to do it with simple clicks or taps.
Check for:
  • Complex gestures: Are there any actions that require complex gestures (e.g., zooming, rotating)?
  • Single pointer alternative: Can you perform the same actions with single clicks or taps?
  • Essential gestures: Are the complex gestures essential to the functionality (e.g., drawing)?

This means that if you accidentally click or tap on something, you should be able to cancel the action or undo it.
Check for:
  • Accidental clicks: Are there any actions that happen immediately when you click or tap?
  • Cancellation options: Can you cancel the action, undo it, or is there a confirmation step?
  • Target size and spacing: Are the clickable areas large enough and spaced out enough to avoid accidental clicks?

This means that the name of a component (like a button) should match the label that you see on the screen. This helps people who use screen readers understand what the component is.
Check for:
  • Component names: Do the names of buttons, form fields, and other components match their visual labels?
  • Screen reader test: Does a screen reader announce the correct name for each component?

This means that if you can do something on the website by moving your device or yourself (like shaking your phone), you should also be able to do it with buttons or other controls on the screen. You should also be able to turn off motion controls if you want to.
Check for:
  • Motion controls: Are there any features that use motion controls?
  • Alternative controls: Can you do the same thing with on-screen controls?
  • Disable motion: Can you turn off the motion controls?
  • Exceptions: Is the motion essential to the functionality (e.g., a game that requires tilting)?

This means that clickable areas should be big enough for people to easily tap or click on them, especially on touchscreens.
Check for:
  • Target size: Are buttons, links, and other interactive elements at least 44 by 44 pixels?
  • Exceptions: Does the target fit into any of the exceptions (equivalent target, inline, user agent control, essential)?

This means that the website shouldn't stop you from using different input methods (like a mouse, keyboard, or touchscreen) at the same time, unless it's necessary for security or user preferences.
Check for:
  • Input restrictions: Are there any limitations on using different input methods together?
  • Essential restrictions: Are the restrictions essential for security or user settings?
